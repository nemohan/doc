# 客户端端口耗尽

最近遇到一个服务访问某个http接口，遇到"can not assign requested address"的问题，网上查了一下，大概率是本地端口耗尽。容器内没有相关的网络工具，查看不方便。可以通过进入容器内进程命名空间的方式来查看。经过排查确实是被访问的服务上有大量处于ESTABLIS状态的连接。

## 查看容器内进程连接数

若容器内有相关的网络工具如netstat，则可以直接使用。若没有则可以在宿主机上，通过进入容器进程命名空间的方式，查看连接数

### 获取进程id

~~~
docker inspect -f {{.State.Pid}} nginx
~~~

### 进入进程的namespace

~~~
nsenter -n -t pid
~~~
