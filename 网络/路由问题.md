有同事再做网络流量解析功能时，遇到一个奇怪的问题。他在一台虚拟机上配置了两个网卡ens160(192.168.40.120)、ens192(192.168.40.33)，两个网卡的ip地址都在同一个网段。当指定源ip为40.120时，出去的数据包经过了网卡ens192，而进来的数据包则经过了ens160，即同一个会话的流量经过了不同的网卡。导致收包解析出错。同事有点懵逼，找我帮忙看看。看了一下系统的路由配置，猜测大概是路由匹配导致的，两个网卡的下一跳地址(网关)都是40.1，匹配时以目的地址进行匹配，大概率匹配了第一个路由规则，其网卡是ens192。

ip route 输出结果:

~~~
default via 192.168.40.1 dev ens192 proto static 
default via 192.168.40.1 dev ens160 proto static 
192.168.40.0/24 dev ens192 proto kernel scope link src 192.168.40.33 
192.168.40.0/24 dev ens160 proto kernel scope link src 192.168.40.120 
~~~

ip rule 输出结果如下:

~~~
0:	from all lookup local
32766:	from all lookup main
32767:	from all lookup default
~~~
