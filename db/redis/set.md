## 存储结构 
若待添加到集合的值能够以整型表示，则创建编码方式为REDIS_ENCODING_INTSET的对象(intset)来存储该值。否则以编码方式为REDIS_ENCODING_HT的对象(哈希表)来存储该值。


### intset存储结构
inset的实现在intset.c文件中。h

```c
typedef struct intset {
    uint32_t encoding; //当前用于编码的每个整型使用的字节数
    uint32_t length; // 集合内元素个数，而不是contents的长度
    int8_t contents[];
} intset;

#define INTSET_ENC_INT16 (sizeof(int16_t))
#define INTSET_ENC_INT32 (sizeof(int32_t))
#define INTSET_ENC_INT64 (sizeof(int64_t))

```

内存结构:
元素按升序存放
```
---------------------------------------------------------------
encoding | length | 元素1 | 元素2 | ...| 元素n
---------------------------------------------------------------
```

创建intset时, encoding的值为INTSET_ENC_INT16(2)

#### 添加元素
确定添加的值能用几个字节表示，目前支持2、4、8字节整型。根据新的值的字节数是否超过当前存储整型用的字节数分为两种情况:
1) 若当前值使用的字节数超过intset->encoding表示的字节数时，则进行升级操作。升级操作时，重新分配内存并将intset->contents中的元素，从后往前依次拷贝到新的位置。若新值大于0（说明新插入的值是最大值)，则将其添加到末尾。若小于0(新插入的值是最小值)，则添加到开始位置

2) 若未超过，则利用二分查找确定元素是否存在。若存在，则终止此次操作。若不存在则添加到intset->contents中。插入位置由二分查找失败的位置确定(这点值得学习)

确定整型值用几个字节表示的方法:
1)若整型值超过32位有符号整型的最大或最小值，则用8字节表示
2)若超过16位有符号整型值的最大或最小值，则用4字节
3)未超过16位有符号整型值的最大或最小值，则用2字节

注意: 整型编码只有升级操作，没有降级操作; 每次新增或移除元素都会重新分配内存

相当于一个动态数组
