## AOF

执行AOF持久化必须同时满足以下条件:

* 没有写rdb或aof的子进程在执行
* server.aof_rewirte_perc 不为0，默认值为REDIS_AOF_REWRITE_PERC(100)。可从配置文件修改该值
* server.aof_current_size 超过 server.aof_rewrite_min_size

* (server.aof_current_size * 100/base) - 100 大于等于server.aof_rewrite_perc

### 父进程

* 若有子进程在写aof文件，则退出此次执行
* 创建子进程执行写aof
* 设置server.aof_rewrite_scheduled 为0、server.aof_rewrite_time_start为现在、server.aof_child_pid为子进程id、server.aof_selected_db 为-1.

### 子进程写AOF的过程

* 创建temp-rewriteaof-pid.aof的临时文件
* 写aof文件，对所有db依次执行以下步骤：1）若当前db为空则跳过 2）获取db迭代器 3）写入选择db的命令 4）对数据库中的每个键执行以下步骤： 4.1）若键已经过期，则跳过该键 4.2）写入对应的对象 4.3）若有过期时间则写入过期时间
* 重命名临时文件名称为temp-rewriteaof-bg-pid.aof
* 根据server.aof_fsync确定数据刷盘方式。



### 父进程等AOF子进程结束时执行的任务

* 重新打开子进程创建的临时文件temp-rewriteaof-bg-pid.aof
* 将server.aof_rewrite_buf_blocks中的内容写入上面的aof文件
* 将temp-rewriteaof-bg-pid.aof重命名为server.aof_filename

### 过程

* 子进程会创建一个temp-pid.pid的临时文件，待全部db写入后。再将其重命名为指定的名称
* 写入版本号
* 写入所有db
* 写入文件结束标志和校验和
* 调用fflush、fsync刷盘
* 将server.dirty置0，server.lastsave 设置为当前时间、server.lastbgsave_status 设置为REDIS_OK。当然这写变量的改动对父进程不可见

### 触发写RDB的条件

若同时满足以条件则将键值对写入rdb文件：

- 没有其他子进程正在执行写入rdb或aof的任务
- 改变的键值对数目超过saveparam.changes配置的数目
- 现在到上一次的写盘时间即server.unixtime-server.lastbgsave超过 saveparam.seconds指定的时间间隔
- 现在到上一次尝试写盘的时间间隔超过 REDIS_BGSAVE_RETRY_DELAY，即server.unixtime - server.lastbgsave_try > REDIS_BGSAVE_RETRY_DELAY。或者上次保存成功

### 整型存储方式

若对象的编码是REDIS_ENCODING_INT，首先尝试能否将其以整型方式存储，否则将以字符串形式存储

整型存储方式:

~~~
编码（1字节）| 值(可变字节)
~~~



| 值                    | 编码                                                 | 长度（字节） |
| --------------------- | ---------------------------------------------------- | ------------ |
| -(1<<7) ~ (1<<7)-1    | (REDIS_RDB_ENCVAL <<6) \| REDIS_RDB_ENC_INT8    0xc0 | 2            |
| -(1<<15) ~ (1 <<15)-1 | (REDIS_RDB_ENCVAL <<6) \| REDIS_RDB_ENC_INT16  0xc1  | 3            |
| -(1<<31)~(1<<31)-1    | (REDIS_RDB_ENCVAL <<6) \| REDIS_RDB_ENC_INT32  0xc2  | 5            |
|                       |                                                      |              |

REDIS_RDB_ENCVAL的值是3。REDIS_RDB_ENC_INT8、REDIS_RDB_ENC_INT16、REDIS_RDB_ENC_INT32的值分别是0、1、2

###　字符串的存储方式

~~~
长度| 字符串值
~~~

长度以大端方式存储，长度的存储方式:

| 长度      | 编码                                                   | 占用字节数 |
| --------- | ------------------------------------------------------ | ---------- |
| 小于64    | REDIS_RDB_6BITLEN    低6位保存长度信息，高两位是编码   | 1          |
| 小于16384 | REDIS_RDB_14BITLEN  第一个字节的高两位保存编码信息     | 2          |
| 其他      | REDIS_RDB_32BITLEN  第一个字节是编码，只用到了最高两位 | 5          |
|           |                                                        |            |

### list的存储方式

编码类型为REDIS_ENCODING_LINKEDLIST

~~~
元素个数| 元素1| 元素2 |...|元素n
~~~

每个元素要么以整型方式存储，要么以字符串形式存储



编码类型为REDIS_ENCODING_ZIPLIST:

* 若ziplist的总字节数不超过11，则尝试将其转为整型存储。
* 若开启了压缩模式，即server.rdb_compression且总字节数超过20。则以lzf算法压缩存储
* 以上条件都不满足，则原样写入文件



未压缩模式:

~~~
ziplist总字节数 | ziplist
~~~



### set的存储方式

编码方式为 REDIS_ENCODING_HT

~~~
元素个数|元素1|元素2 | ... | 元素n
~~~



编码方式为REDIS_ENCODING_INTSET

- 若总字节数不超过11，则尝试将其转为整型存储。
- 若开启了压缩模式，即server.rdb_compression且总字节数超过20。则以lzf算法压缩存储
- 以上条件都不满足，则原样写入文件
