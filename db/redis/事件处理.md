## 事件处理循环

分为三种情况:
​    1) 若既没有设置AE_TIME_EVENTS 、也没有设置AE_FILE_EVENTS标志(没有要处理的事件)则什么也不做，立即退出当前执行
​    2) 若最大文件描述符不为-1，或设置了AE_TIME_EVENTS且没有设置AE_DONT_WAIT
​    2.1) 若设置了AE_TIME_EVENTS 且没有设置AE_DONT_WAIT，则查找最先触发的定时任务。若找到了定时任务则计算该定时任务到现在时刻的差值，用于设置等待网络IO的超时时间。若没有找到定时任务，又分为两种情况: 设置了AE_DONT_WAIT, 则等待网络IO的超时时间设置为0(不等待); 否则永远等待
​    2.2) aeApiPool等待网络IO事件

    3) 若设置了AE_TIME_EVENTS,则处理定时任务

### 定时任务

以单链表维护定时任务列表，新的定时任务被放在列表头部.



定时任务相关参数:



处理过程:
1) 检查有没有时钟漂移(clock skew),若有时钟漂移，将所有定时任务的超时时间的秒数部分设置为0,会立即执行所有定时任务

2)忽略定时任务ID超过最大ID的任务(什么情况会发生), 执行超时的定时任务

3)定时任务执行后返回AE_NOMORE(-1) 或下次执行的时间间隔(duration)，由1000/server.hz得到。 返回AE_NOMORE则会删除当前定时任务

server.hz的默认值是REDIS_DEFAULT_HZ(10即每秒执行10次)，控制定时任务的执行频率(每秒执行多少次)。


```c
/* Using the following macro you can run code inside serverCron() with the
 * specified period, specified in milliseconds.
 * The actual resolution depends on server.hz. 
//1000 / server.hz 执行一次的周期.

#define run_with_period(_ms_) if ((_ms_ <= 1000/server.hz) || !(server.cronloops%((_ms_)/(1000/server.hz))))
```

## 过期键值对的处理



* 静态变量current_db保存了上次处理的db、timelimit_exit(上次处理是否是因为到达时间限制而退出)、last_fast_cycle


执行过程:

1） 每次处理REDIS_DBCRON_DBS_PER_CALL(16)个DB。

2）针对每个DB。若当前db没有过期键，跳过当前DB，处理下一个DB

​	2.1)若过期键数目超过DICT_HT_INITAL_SIZE(4)且元素数目不足哈希表大小的1/100，则跳过当前DB。

​	2.2) 一次性处理ACITVE_EXPIRE_CYCLE_LOOKUPS_PER_LOOP（20）随机键值对

​	2.3)每执行16次，检查时间是否超过timelimit。若超过则设置timelimit_exit为1，且结束
