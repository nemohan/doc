两种不同的定义:

方式1：

* 所有叶子节点都处于同一高度
* 非叶子节，根节点除外，包含ceil(n/2)(大于等于n/2的最小整数，ceil(5/2)为3)到n个子节点。根节点可以包含2到n个子节点
* 叶子节点最少包含floor((n-1)/2)（即小于等于 (n-1)/2的最大整数)



假设b+树的fanout是5，即n为5

| 节点     | 子节点个数 | key的个数 |
| -------- | ---------- | --------- |
| 根节点   | 2到5       | 1到4      |
| 内部节点 | 3到5       | 2到4      |
| 叶子节点 |            | 2到4      |



假设b+树的fanout是6

| 节点     | 子节点个数 | key的个数 |
| -------- | ---------- | --------- |
| 根节点   | 2到6       | 1到5      |
| 内部节点 | 3到6       | 2到5      |
| 叶子节点 |            | 2到5      |

方式2：

* 所有叶子节点都处于同一高度 
* 每个节点，根节点除外，包含m个key，其中 d <= m <= 2d。d是b+树的一个参数，被称作b+树的order。根节点可以包含1 <=m <=2d 个key。



假设b+树的fanout是5，则d=2

| 节点     | 子节点个数 | key的个数 |
| -------- | ---------- | --------- |
| 根节点   | 2到5       | 1到4      |
| 内部节点 | 3到5       | 2到4      |
| 叶子节点 |            | 2到4      |



假设b+树的fanout是6，则2d=5，则d是取顶还是取底

取顶 d=3

取底d=2



| 节点     | 子节点个数            | key的个数      |
| -------- | --------------------- | -------------- |
| 根节点   | 2到6       2到6(取底) | 1到5      1到5 |
| 内部节点 | 4到6      3到6（取底) | 3到5     2 到5 |
| 叶子节点 |                       | 3到5     2到5  |



a按照这种方式定义，显然m不能是奇数

上面的定义出自《Database management system》第三版
