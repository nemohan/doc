## 关键的结构体定义及用途

### keyType 

keyType表示key的类型，由两个常量取值:

* keyTypeDel 表示删除
* keyTypeVal 表示增加

~~~go
type keyType uint

// Value types encoded as the last component of internal keys.
// Don't modify; this value are saved to disk.
const (
	keyTypeDel = keyType(0)
	keyTypeVal = keyType(1)
)
~~~



### internalKey

internalKey由两部分组成: key 和 8字节的64位整数，其中低8位为key的类型即keyType,高54位为序列号。

序列号的来源, 由db维护全局的seq。

internalKey作为key在memDB中的存储形式

### batch

Batch结构体的定义

~~~go

type Batch struct {
	data  []byte
	index []batchIndex

	// internalLen is sums of key/value pair length plus 8-bytes internal key.
	internalLen int

	// growLimit is the threshold in order to slow down the memory allocation
	// for batch when the number of accumulated entries exceeds value.
	//
	// batchGrowLimit is used as the default threshold if it's not configured.
	growLimit int
}

~~~

batchIndex的定义如下

* keyType 表示key-value的操作类型是插入还是删除
* keyPos、keyLen分别表示key在batch中存储的位置及长度
* valuePos、valueLen表示value在batch中存储的位置及长度

~~~go
type batchIndex struct {
	keyType            keyType
	keyPos, keyLen     int
	valuePos, valueLen int
}
~~~

key-value在batch中的存储方式:

~~~
keyType（1字节) | key 长度 (可变字节数) | key | value长度(可变字节数) | value
~~~

batch用来批量写入key-value。batch用一个大的字节切片来存储多个key-value，使用batchIndex记录每个key、value在切片中的位置以方便使用。

~~~
// Must hold RW-lock if prev == true, as it use shared prevNode slice.
func (p *DB) findGE(key []byte, prev bool) (int, bool) {
	node := 0
	h := p.maxHeight - 1
	for {
		//下一个元素的索引位置
		next := p.nodeData[node+nNext+h]
		cmp := 1
		if next != 0 {
			//o 是key-value在 kvData的偏移位置
			o := p.nodeData[next]
			//p.nodeData[next + nKey]是key的长度, next+nKey是key所在的位置
			cmp = p.cmp.Compare(p.kvData[o:o+p.nodeData[next+nKey]], key)
		}
		// 小于key
		if cmp < 0 {
			// Keep searching in this list
			node = next
		} else {
			//TODO:若此时h !=0, cmp == 0, 之所以继续往下层走就是为了记录前向节点
			//大于等于key
			if prev {
				//记录当前level的此节点的前一个节点
				p.prevNode[h] = node
			} else if cmp == 0 {
				return next, true
			}
			if h == 0 {
				return next, cmp == 0
			}
			h--
		}
	}
}
~~~



~~~go
// Put sets the value for the given key. It overwrites any previous value
// for that key; a DB is not a multi-map.
//
// It is safe to modify the contents of the arguments after Put returns.
func (p *DB) Put(key []byte, value []byte) error {
	p.mu.Lock()
	defer p.mu.Unlock()

	if node, exact := p.findGE(key, true); exact {
		kvOffset := len(p.kvData)
		p.kvData = append(p.kvData, key...)
		p.kvData = append(p.kvData, value...)
		p.nodeData[node] = kvOffset
		m := p.nodeData[node+nVal]
		p.nodeData[node+nVal] = len(value)
		p.kvSize += len(value) - m
		return nil
	}

	h := p.randHeight()
	if h > p.maxHeight {
		for i := p.maxHeight; i < h; i++ {
			p.prevNode[i] = 0
		}
		p.maxHeight = h
	}

	kvOffset := len(p.kvData)
	p.kvData = append(p.kvData, key...)
	p.kvData = append(p.kvData, value...)
	// Node
	node := len(p.nodeData)
	p.nodeData = append(p.nodeData, kvOffset, len(key), len(value), h)
	//要更新当前节点的前向节点，

	//假如前一个节点只有一层，当前节点有三层
	//类比单向链表分析
	for i, n := range p.prevNode[:h] {
		//i 不超过11
		//为什么n + nNext +i , n不就是当前节点的前一个节点么
		m := n + nNext + i
		//前一个节点的下一个节点，成为当前节点的下一个节点
		p.nodeData = append(p.nodeData, p.nodeData[m])

		//前一个节点的下一个节点指向当前节点
		p.nodeData[m] = node
	}

	p.kvSize += len(key) + len(value)
	p.n++
	return nil
}
~~~
