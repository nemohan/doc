## 使用多阶段构建(multi-stage builds)

`多阶段构建`需要Docker17.05或更高版本

`多阶段构建`支持在一个Dockerfile中使用多个FROM指令,每个FROM指令可以使用不同的基础镜像。

~~~dockerfile
FROM golang:1.7.3
WORKDIR /go/src/github.com/alexellis/href-counter/
RUN go get -d -v golang.org/x/net/html  
COPY app.go .
RUN CGO_ENABLED=0 GOOS=linux go build -a -installsuffix cgo -o app .

FROM alpine:latest  
RUN apk --no-cache add ca-certificates
WORKDIR /root/
COPY --from=0 /go/src/github.com/alexellis/href-counter/app .
CMD ["./app"]  
~~~

注意，上面的Dockerfile中的第10行的COPY --from=0，将第一个阶段的构建结果拷贝到当前构建过程



### 命名构建阶段

每个FROM是一个独立的构建阶段，默认编号从0开始（自上到下）。可以给构建阶段提供名称

~~~dockerfile
FROM golang:1.7.3 AS builder
WORKDIR /go/src/github.com/alexellis/href-counter/
RUN go get -d -v golang.org/x/net/html  
COPY app.go    .
RUN CGO_ENABLED=0 GOOS=linux go build -a -installsuffix cgo -o app .

FROM alpine:latest  
RUN apk --no-cache add ca-certificates
WORKDIR /root/
COPY --from=builder /go/src/github.com/alexellis/href-counter/app .
CMD ["./app"] 
~~~



### 只构建部分阶段

下面的命令，是构建到builder阶段还是只构建builder阶段

~~~
$ docker build --target builder -t alexellis2/href-counter:latest .

~~~



### 使用外部镜像作为构建阶段
